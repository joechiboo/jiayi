# 井字遊戲 - 專案開發憲章

## 開發原則

### 1. 簡潔優先
- 使用原生 JavaScript，不使用框架或建置工具
- 單一 HTML 檔案架構（all-in-one 方式）
- 最少依賴 - 優先使用原生瀏覽器 API

### 2. 程式碼品質標準
- 撰寫乾淨、可讀且有良好註解的程式碼
- 使用有意義的變數和函式名稱
- 遵循一致的程式碼格式
- 維持關注點分離（遊戲邏輯、UI、AI）

### 3. 使用者體驗一致性
- 符合現有遊戲集合的視覺設計
- 使用一致的配色方案和樣式模式
- 確保所有裝置的響應式設計
- 為所有互動提供清楚的視覺回饋

### 4. AI 設計哲學
- 實作多種難度級別（簡單、中等、困難）
- 簡單：隨機有效移動
- 中等：阻擋玩家獲勝 + 基本策略
- 困難：使用 Minimax 演算法進行最佳遊玩
- AI 難度應該可在遊戲開始前或遊戲中選擇

### 5. 測試與驗證
- 部署前手動測試所有遊戲情境
- 徹底驗證勝負/平手偵測邏輯
- 在多種螢幕尺寸和裝置上測試
- 確保 AI 決策無錯誤

### 6. 效能要求
- 對使用者互動立即回應
- AI 移動應在 100ms 內完成
- 避免不必要的 DOM 操作
- 流暢的動畫效果（如有）

### 7. 無障礙設計指南
- 使用語意化 HTML 元素
- 確保鍵盤導航支援
- 提供清楚的遊戲狀態提示
- 維持良好的色彩對比度

### 8. 與現有專案整合
- 遵循與其他遊戲相同的檔案結構（2048/、snake/ 等）
- 重複使用共用 UI 元件（返回按鈕、Claude 標誌）
- 更新主 index.html 以連結到井字遊戲
- 維持一致的導航模式

## 技術限制

### 必要技術棧
- HTML5 用於結構
- CSS3 用於樣式（不使用預處理器）
- 原生 JavaScript（ES6+）用於邏輯
- 不使用外部函式庫或框架

### 瀏覽器支援
- 現代瀏覽器（Chrome、Firefox、Safari、Edge）
- 行動瀏覽器（iOS Safari、Android Chrome）
- 不需要支援 IE11
