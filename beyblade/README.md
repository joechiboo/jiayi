# ⚡ 戰鬥陀螺 Beyblade Battle

雙人同機對戰的戰鬥陀螺遊戲！選好陀螺、衝高轉速，在競技場上一決勝負！

## 🎮 遊戲流程

1. **1P 盲選陀螺** — 選完交給 2P（遮罩畫面，不讓對方偷看）
2. **2P 盲選陀螺**
3. **公開雙方陀螺** — 心理戰揭曉！
4. **決定轉速** — 兩種輸入方式任選：
   - ⏱️ **時機型**：指針來回擺動，按在紅區拿高轉速
   - 🔥 **連打型**：限時連打，打越多轉越快
5. **自動戰鬥** — 兩顆陀螺互鬥，全程看戲！

## ⚔️ 雙層剪刀石頭布

### 屬性三角
🔥 火 > 🌿 草 > 💧 水 > 🔥 火

### 類型三角
- ⚔️ **攻擊型** 克 持久型 — 趁它沒防禦時撞飛（移動範圍大、會衝刺、攻擊力高，但轉速消耗最快）
- 🛡️ **防禦型** 克 攻擊型 — 彈開攻擊、消耗它（守住中央、抗爆體，但攻擊最弱）
- ⏳ **持久型** 克 防禦型 — 耗到防禦型先停（轉速衰減最慢，磨耐力）

屬性 × 類型雙層克制，選陀螺就是心理戰！

## 🏆 勝負條件

1. **停轉敗北** — 先停止旋轉的輸
2. **出界敗北** — 被撞出場外的輸
3. **爆體敗北** — 爆體槽滿了直接爆炸！

## 🎲 轉速的賭博感

轉速衝越高：
- ✅ 轉得越久、撞人越痛
- ❌ 亂竄幅度變大、越容易噴出場外
- 💥 雙方都高轉速正面對撞 → 爆體槽暴增（被克屬性爆得更快）

## 💥 漫畫演出

- **殘影拖尾** — 轉速越高殘影越長
- **定格打擊感** — 大力對撞畫面定格 + 白色集中線閃光
- **狀聲字** — 「轟!!」「鏗!」「咻—」「飛出去啦!!」「爆體!!!」
- **決勝時刻!!** — 22 秒後紅圈脈動、轉速加速衰減，30 秒內必分勝負

## 🗺️ 四種場地（每場開戰隨機抽）

場地由亂數種子推導，連線兩端必然一致；再戰會換場地。

| 場地 | 特性 |
|---|---|
| 🏟️ 標準競技場 | 均衡的碗型場地 |
| 🌋 火山口 | 中央隆起把陀螺往外推，防禦型無法龜中間 |
| 🧊 冰晶湖面 | 超滑（阻尼減半）、擊退滑很遠，但冰冷讓轉速衰減變慢 |
| 🎯 迷你鬥場 | 場地縮小 22%，貼身肉搏碰撞不斷 |

平衡數據與調參方式見 [BALANCE.md](BALANCE.md)（附無頭模擬驗證工具 `tools/bb-sim.js`）。

## 🗺️ 場地物理（簡化版碗型競技場）

- 陀螺自然滑向中心
- 轉速越高離心移動越大（易噴出）
- 碰撞互扣轉速：克制方扣得少、被克方扣得多
- 全體繞場軌道運動，轉速越高繞越快
- 雙防禦對局：傷害有轉速比加成，轉速高的磨贏

## 🌐 連線對戰

雙方各拿一支手機遠端對決！

**玩法**：一方按「建立房間」拿到 4 位數代碼 → 另一方輸入代碼加入 → 各自秘密選陀螺 → 各自發射 → 兩邊同步看同一場戰鬥。

**架構**（參考 [Fast-Trivia](https://github.com/joechiboo/Fast-Trivia) 的模式）：
- 後端：`server/` 資料夾，Node.js + Express + Socket.IO，部署在 Render.com
- 前端：GitHub Pages（本頁面），偵測到非 localhost 就連 Render 伺服器
- 同步方式：**確定性模擬**。伺服器只撮合「雙方選擇 + 轉速 + 亂數種子」，
  戰鬥由兩邊客戶端用同一種子、固定時間步長（1/120s）各自演算，
  演出完全一致、幾乎不吃流量，免費方案綽綽有餘

### 本機開發

```bash
cd beyblade/server
npm install
npm run dev        # http://localhost:3000
```

再用任一靜態伺服器開前端（例如在 jiayi 根目錄 `python -m http.server 8080`），
開兩個瀏覽器分頁連 http://localhost:8080/beyblade/ 就能自己跟自己對戰測試。

### Render 部署設定

1. Render → New → Web Service → 連接 `joechiboo/jiayi` repo
2. **Name**: `jiayi-beyblade`（前端寫死連 `https://jiayi-beyblade.onrender.com`，取別的名字就要改 index.html 裡的 `SERVER_URL`）
3. **Root Directory**: `beyblade/server`
4. **Build Command**: `npm install`
5. **Start Command**: `npm start`
6. 環境變數：`NODE_ENV=production`（`CLIENT_URL` 可選，預設已允許 joechiboo.github.io）
7. Free 方案即可；閒置後喚醒約需 1 分鐘，前端有做等待提示

## ⚡ 必殺時機（QTE）

戰鬥中偵測到「即將大力對撞」時（開戰 3 秒後），畫面定格、跳出必殺時機：

- 時機條限時 5 秒，按在紅心＝超必殺
- 按得準：這一撞攻擊倍率最高 2.6 倍；防守方按得準也能大幅減傷
- 沒按到＝miss，普通碰撞；每場只有一次機會
- 本機版：定格時 1P/2P 各按自己的時機條；連線版：各自在自己手機上按
- 同步原理：觸發條件只依模擬狀態（兩端同一步定格），結果經伺服器湊齊後
  從同一步繼續確定性模擬，延遲只影響定格時長、不影響演出一致性
