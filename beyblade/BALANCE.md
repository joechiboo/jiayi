# ⚖️ 戰鬥平衡驗證文檔

用無頭模擬（headless simulation）量化驗證戰鬥平衡：把 `index.html` 的遊戲程式碼載入
Node.js vm 直接跑戰鬥迴圈，每個「場地 × 陀螺組合」跑 480 場（120 種子 × 4 種轉速配置），
統計戰鬥長度、必殺觸發率與死因。**改任何戰鬥參數前後都應重跑一次對照。**

```bash
node beyblade/tools/bb-sim.js
```

## 🎯 平衡目標

| 指標 | 目標 | 理由 |
|---|---|---|
| 中位戰鬥長度（模擬時間） | 火系組合 4~10 秒、耐久組合 15~30 秒 | 實際體感另加 3.5s 倒數＋5s 必殺定格 |
| 必殺(QTE)觸發率 | 慢速對局 100%、快攻對局 ≥74% | 「幾乎每場都用得到必殺」是核心體驗 |
| QTE 後平均餘命 | ≥2 秒 | 必殺不該是「觸發即結束」 |
| 死因分布 | 三種死法（停轉/出界/爆體）都要出現 | 單一死因＝機制失效 |

## 🎛️ 參數旋鈕（都在 index.html）

| 旋鈕 | 位置 | 目前值 | 效果 |
|---|---|---|---|
| 開場漸進傷害 | `ramp` in update() 碰撞區 | `0.35+0.1t`（6.5s 滿） | 防開場暴斃 |
| 碰撞傷害 | `base=12+rel*1.9` | 以撞速為主 | 貼身互磨不痛 |
| 爆體單次上限 | `Math.min(uAtk>1.5?45:24, ...)` | 24／必殺 45 | 大撞 4~5 次才爆 |
| 爆體撞速係數 | `relScale=min(1,rel/80)` | rel<80 打折 | 互磨不累積爆體 |
| 擊退上限 | `Math.min(uAtk>1.5?260:170, ...)` | 170／必殺 260 | 只有必殺能撞飛 |
| 站穩期 | `escape` 的 `40*(1-t/8)` | 8 秒漸退 | 開場不易出界 |
| QTE 起始/門檻 | update() 開頭 | 3s 起，7/10/14s 降門檻，16s 強制 | 保證觸發 |
| QTE 視窗 | `QTE_WINDOW` | 5 秒 | 伺服器逾時須 > 視窗（現 8s） |
| 必殺倍率 | applyQte() | 攻 1+1.6q、防 1+1.1q、4s 內首撞 | q=時機準度 0~1 |
| 場地參數 | `ARENAS` | R/damp/escapeAdd/decayMul/hill | 新場地加這裡 |

## 📜 調參紀錄

**2026-07-11（初版問題 → 修正）**
- 症狀：火系對局中位 2.0~2.2 秒即結束、水鏡像 3.7 秒互磨爆體、必殺常常來不及用（水鏡像觸發率 0%）
- 元兇 1：`burst += dmg*0.32` 無上限 → 一次衝刺對撞直接爆體 → 加上限 24 + 撞速係數
- 元兇 2：擊退無上限（速度 200+）→ 開場被撞飛出界 → 上限 170（必殺 260 保留撞飛）
- 元兇 3：貼身互磨（rel 小）傷害含常數項 45 → 改 `12+rel*1.9` 以撞速為主
- 場地版問題：冰面低護欄+滑 → 火系 81% 五秒內出界 → 護欄改 +15；
  同向迴旋鏡像在火山/冰面繞圈追不上彼此 → QTE 16 秒強制觸發＋套用後互衝 60
  （互衝一度設 110，火水對局 41% 五秒內結束，降為 60）
